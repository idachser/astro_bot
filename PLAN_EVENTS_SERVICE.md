# План: собственный сервис расчёта небесных событий с API

## Цель

Заменить внешний iCal-фид in-the-sky.org собственным сервисом, который
рассчитывает события астрономически (эфемериды, без сети) и отдаёт их по
HTTP API. Сервис разрабатывается **сразу в отдельном репозитории**
(рабочее имя — `skyevents`); этот репозиторий остаётся репозиторием
бота, связь между ними — только API-контракт и Docker-образ.

Побочные выгоды:

- снимается зависимость от неофициального фида без SLA и лицензионное
  ограничение «некоммерческое использование с атрибуцией»;
- события получают структурированные параметры (тип, тела, расстояние,
  элонгация…) вместо готового английского текста → тексты можно
  рендерить шаблонами на любом языке (русская локализация бота);
- становится реализуемым отложенный пункт PLAN.md — фильтр видимости
  события с локации пользователя (высота светила считается из тех же
  эфемерид).

## Ключевые решения

- **Движок: Skyfield + эфемериды JPL DE440s** (`de440s.bsp`, ~32 МБ,
  покрывает 1849–2150). Skyfield — чистый Python (MIT), работает офлайн,
  в `skyfield.almanac` уже есть фазы Луны, сезоны, противостояния и
  соединения; `skyfield.searchlib` (find_minima/find_maxima/find_discrete)
  закрывает остальные поиски. Альтернативы отклонены: PyEphem устарел,
  Astropy — слишком низкоуровневый для поиска событий.
- **Событие = данные, не текст.** Модель:
  `uid` (детерминированный: `{type}:{тела}:{дата}`), `type`,
  `dt_utc` (момент пика, ISO-8601), `bodies`, `params` (JSON: разделение
  в градусах, дистанция в км, элонгация, звёздная величина…).
  Тексты (`summary`/`description`, en + ru) рендерятся шаблонами по типу —
  на стороне сервиса, параметром `lang`.
- **Отдельный репозиторий с первого коммита.** Никаких импортов между
  ботом и сервисом; бот знает только `EVENTS_API_URL`. Деплой бота
  подключает сервис как Docker-образ из registry (GHCR), который
  собирает и публикует CI сервиса.
- **Python 3.12+, FastAPI + uvicorn.** Ограничение «3.11 из-за aiogram 2»
  касается только бота; сервис от него свободен.
- **Хранилище сервиса**: собственный SQLite-кеш «год → события».
  Генерация года выполняется фоном при старте и по расписанию (текущий
  год + следующий в декабре, как сейчас в боте); API читает только из
  кеша и явно сообщает границы покрытия.

## Этап 2. Генераторы событий

Каждый генератор — функция `год → list[Event]`, покрытая тестами с
допусками из этапа 0. По убыванию ценности для бота:

- [ ] Фазы Луны (новолуние, четверти, полнолуние) — `almanac.moon_phases`.
- [ ] Равноденствия и солнцестояния — `almanac.seasons`.
- [ ] Перигей/апогей Луны — экстремумы расстояния Земля–Луна.
- [ ] Противостояния и соединения планет с Солнцем, нижние/верхние
      соединения Меркурия и Венеры — `almanac.oppositions_conjunctions`.
- [ ] Наибольшие элонгации Меркурия и Венеры — максимумы элонгации.
- [ ] Сближения — минимумы углового разделения; генерируем только close
      approach (минимум), отдельное событие «conjunction» по равенству
      прямых восхождений, как у in-the-sky, не дублируем. Пороги — по
      классам пар: Луна–планета ≈7° (как `maxdiff=7` в фиде),
      планета–планета ≈2.5° — единый порог 7° завалил бы ленту
      неинтересными «сближениями» планет; уточнить по диффу этапа 0.
- [ ] Лунные затмения — `skyfield.eclipselib.lunar_eclipses`
      (модуль помечен экспериментальным — закрепить версию Skyfield
      в lockfile и накрыть тестами на известных затмениях).
- [ ] Солнечные затмения — поиск в окрестности новолуний по угловому
      разделению Солнце–Луна; порог — сумма видимых радиусов **плюс
      горизонтальный параллакс Луны** (~0.95°), иначе геоцентрический
      критерий пропустит большинство частных затмений, видимых только
      с части поверхности Земли. Тип (полное/кольцеобразное/частное) —
      по геометрии.
- [ ] Метеорные потоки — статический каталог (рабочий список IMO:
      название, солнечная долгота пика, ZHR, радиант); момент пика в
      конкретном году считается из солнечной долготы.
- [ ] (Опционально, фаза 2) Покрытия Луной ярких звёзд и планет,
      перигелий/афелий Земли, максимальный блеск Венеры.

Вне скоупа: кометы и противостояния астероидов — требуют регулярно
обновляемых орбитальных элементов (MPC); вернуться после запуска.

## Этап 4. API

- [ ] `GET /events?from=YYYY-MM-DD&to=YYYY-MM-DD&types=...&lang=ru|en`
      — основной эндпоинт (полуинтервал по UTC). Валидация:
      `from <= to`, ограничение ширины окна (например, ≤ 400 дней).
- [ ] Явный контракт покрытия: кеш хранит только сгенерированные годы,
      и «пусто, потому что не считали» не должно выглядеть как «событий
      нет» — либо поле `coverage` в ответе, либо 422 при запросе за
      пределами сгенерированного диапазона.
- [ ] `GET /health` — для compose healthcheck и CI smoke-теста.
- [ ] (Опционально) `GET /calendar.ics?year=` — совместимость с форматом
      текущего фида: первый шаг переключения бота сводится к подмене
      URL фида (сейчас `ICS_FEED_URL` захардкожен в `config.py` — вынести
      в `.env`, это единственная правка кода бота на этом шаге).
- [ ] Генерация кеша — фоном: `/health` отвечает сразу после старта
      (попарные поиски сближений могут занимать минуты), регенерация по
      расписанию. Версия генератора хранится **в кеше**, не в uid:
      смена версии перегенерирует кеш, а uid остаются детерминированными
      от сущности события — иначе каждое обновление логики меняло бы все
      uid и плодило дубликаты в upsert-таблице бота.

## Риски

- **Состав событий не совпадёт с фидом один в один**: in-the-sky
  добавляет события с ручной курацией (яркость, наблюдаемость). Принять:
  наш состав определяется списком генераторов и порогами; полный дифф
  этапа 0 покажет, чего заметно не хватает.
- **Точность**: моменты сближений зависят от того, считаем ли
  геоцентрически. Считать геоцентрически (как фид), допуски зафиксировать
  тестами.
- **Два репозитория — контракт надо версионировать**: изменения формата
  ответа API сопровождать версией (заголовок или `/v1/` в пути с самого
  начала), чтобы деплой сервиса не ломал бота.
- **Размер эфемерид** (~32 МБ) раздувает образ — приемлемо; при
  необходимости сузить период через `python -m jplephem excerpt`.
