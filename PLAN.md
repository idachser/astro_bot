# План: переход с парсинга astronomy.com на iCal-фид in-the-sky.org

## Цель

Заменить хрупкий скрейпинг HTML-статьи «Sky This Week» на структурированный
источник — iCal-календарь событий in-the-sky.org:

```
https://in-the-sky.org/newscalyear_ical.php?year=<YYYY>&maxdiff=7
```

Фид отдаёт ~500 событий в год (фазы Луны, соединения, противостояния,
покрытия, метеорные потоки, затмения) с точным UTC-временем, заголовком,
описанием и ссылкой. Данные © Dominic Ford, использование — некоммерческое
с обязательной атрибуцией.

## Этап 1. Новый источник данных

  на новую схему (этапы 2–3).

## Этап 2. Схема БД

- [x] Новая таблица `events`:
      `uid TEXT PRIMARY KEY, dt_utc TEXT NOT NULL, summary TEXT NOT NULL,
      description TEXT, url TEXT` — вместо строковой даты «Friday, July 3»
      хранить ISO-дату/время.
- [x] Переписать `db_queries.py` на параметризованные запросы
      (убрать f-string SQL), добавить выборку по диапазону дат:
      `WHERE date(dt_utc) BETWEEN ? AND ?`.
- [x] `services/events.py`: `sync_events()` (upsert по uid),
      `get_events_on_day()` / `get_events_between()` по ISO-датам.
      Старые `write_events()` / `get_events()` оставлены как заглушки
      до перевода хендлеров (этап 3).
- [x] Миграция: `init_storage()` при старте создаёт таблицы и дропает
      старую таблицу events (без колонки uid); вызывается из `__main__`.

## Этап 3. Хендлеры

- [x] `today` / `yesterday` / `tomorrow`: убрана regex-обрезка нулей и
      `DATE_FORMAT`-строки, запрос по `date.today() ± timedelta`;
      несколько событий за сутки склеиваются в одно сообщение.
- [x] `week`: листалка `<` / `>` по дням недели, в callback_data —
      ISO-дата дня (`week_2026-07-03`); глобальный счётчик удалён.
- [x] `specific_date`: парсит «July 15» в дату, на нераспознанный ввод
      отвечает подсказкой о формате вместо падения.
- [x] `autosend_events`: по субботам `sync_events()` + однострочный
      дайджест недели с недельной клавиатурой; `get_users_ids()` починен,
      ошибки отправки (заблокировавшие бота юзеры) не роняют рассылку.
- [x] Дополнительно: `sync_events()` вызывается при старте бота, чтобы
      свежая база наполнялась сразу, а не в ближайшую субботу.

## Риски

- Фид неофициальный, без SLA: при недоступности — оставлять старые данные
  в БД и логировать, бот продолжает отвечать из кеша.
- Описания на английском: типовые события (фазы Луны и т.п.) позже можно
  локализовать шаблонами по ключевым словам summary.
